Frontend at `http://localhost:5173/`, backend at `http://localhost:5000/api/...`.
- Verification approach used throughout: lint (`npm run lint`) + build (`npm run build`) on the client, manual curl exercises against a live server for every backend endpoint, and for frontend data-layer logic (date math, slot mapping, conflict-decision branches) replicating the exact JS logic in a throwaway Node script against the real live backend rather than against mocks. This catches real bugs in the actual algorithms even without a working browser. Real rendered-DOM/interaction verification (does it look right, does dragging actually highlight cells) is NOT possible in this environment without sudo (see Blockers) — those pieces were verified by the user manually, or are still pending manual verification.
