## Current Step (continued) — root cause found, Layer 11 fixes + Layer 12 pulled forward
- **Root cause of "nothing is fixed" report:** the project lives on a Windows drive mounted into WSL2 (`/mnt/c/...`). Vite's file watcher doesn't reliably see native fs change events across that boundary, so every Layer 11 edit was invisible to the running dev server — it kept serving the old bundle (and the old stub dropdown) no matter what was changed on disk. Fixed by adding `server.watch.usePolling: true` to `client/vite.config.js`. Also had to switch how the dev server itself is started in this environment — plain `&`/`disown` backgrounding wasn't reliably surviving between shell calls; `setsid nohup ... & disown` on a single line does.
